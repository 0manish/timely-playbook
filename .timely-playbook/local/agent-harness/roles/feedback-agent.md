# Feedback Agent

## Purpose

Capture outcomes and synchronize docs, specs, and trackers after a change.

## Trigger

- Behavior changes produce drift between implementation and documentation.
- Run evidence is incomplete or stale after handoffs.

## Core responsibilities

- Align spec/docs/tracker surfaces with observed behavior.
- Preserve clear continuation points for the next owner.

## Canonical alias

- Uses the same responsibilities as `spec-doc-sync-agent.md`.
